Description:

Welcome to the new GoneGamer website! We proved that the website is capable of some cool things in version 1, but we all know it's capable of so much more! And this time we're going to do it right! That's right! This time, we're taking away the tedium for hand writing every element and embracing both templating (through laravel's blades) and automation (through Gulp)! This should lay the groundwork for letting users help build the universes they want through simple meta-page creation. We still have a ways to go, but we're off to a good start. Read on and rock on!

Run Local Development

  • Only if not currently set up:
    • Install and Vagrant and VirtualBox
    • In terminal, in project directory:
      • vagrant box add laravel/homestead
    • Install Homestead:
      • composer global require "laravel/homestead=~2.0"
      • Add: ~/.composer/vendor/bin to Path so homestead command is global
      • homestead init
    • If on Windows, may need to install the following:
      • Install Python
        • Only need to follow instructions through including python variable in path
      • Install Wampserver for the php executable
        • You'll need this during the composer install.
        • Find the php.ini file (should be in: conf/php/5.6.X) and enable OpenSSL, and curl options at the least.
        • Use the highest available php for best results.
      • Install OpenSSL
      • Install Composer
        • Installing python, php, and OpenSSL were all the pre-requisites for this.
        • When installing composer, you'll most likely have to choose a php.exe file, make sure to use the php.exe for the php.ini version you set up before.

    Edit /etc/hosts: 192.168.10.10 appserv4.app
    "homestead edit" - map: appserv4.app to: /home/vagrant/Code/appserv4/public
    homestead up OR vagrant up
    gonegamer.app/ || gonegamer.app/test

Install

More detailed instructions can be found in our wiki, here. After the general set up is complete, you should be able to run the following install commands:


    npm i
    bower i
    composer install
    php artisan key:generate

    vagrant up
    vagrant ssh
    cd gonegamer
    php artisan migrate
    php artisan db:seed

    (Note: i is terminal accepted shorthand for "install")

Build and Publish

This project uses gulp to build the working files using the tasks defined in the /gulpfile.js


    gulp
    gulp watch (allows autocompiling of js/css resrouces on save and autoreload if autoreloader plugin installed in browser.)
    # gulp --production

Info

List of current plug-ins: copy, SASS, uglify, watch

Copy: Copies files directly from one directory to another - useful for moving files that don't need to be compiled, concatenated, uglified, etc. Currently moves external libraries from either bower or node modules into the appropriate resources folder

Sass: Processes multiple .scss documents into a single CSS file REQUIRES referenced source to be list of @import of all other necessary files Currently uses the resources/assets/scss/app.scss file as import source and processes referenced files into a single public/css/app.css file

Uglify: Takes any javascript file, minifies it, replaces function and variable names with simple characters and outputs it to a selected destination Currently uses the concatenated build/js/<%= pkg.name %>.js file built from all javascript files in the source and outputs the dist/js/<%= pkg.name %>.min.js

Watch: Opens a socket in the command line that looks for designated changes Currently looks for changes made to either src//*.html files and runs them through htmlhint or changes made to src//*.js files and runs them through jshint Can be expanded to automatically clean, copy, less, cssmin, htmlhint:beforecat, jshint:beforecat, cooncat, uglify, htmlhint:aftercat, jshint:aftercat -> exec:dev
